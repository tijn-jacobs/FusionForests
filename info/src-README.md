# src/ — C++ backend

## Active implementation

The files in this directory implement the **Stan-based BART engine**, which is the sole backend used in FusionForests. The name "Stan" here is a historical convention borrowed from the original BART literature (Chipman, George & McCulloch) and has nothing to do with the Stan probabilistic programming language.

### File overview

| File(s) | Role |
|---|---|
| `Prerequisites.h` | Central include: standard headers, Rcpp, `Random.h`, math constants, `standard_normal_cdf/quantile` |
| `Random.h` | C-based random number generator |
| `Info.h` | Lightweight data containers (`DataInfo`, `PriorInfo`) used by the Stan engine |
| `StanTree` | A single decision tree node with fixed-variance leaf prior |
| `StanTreeFunctions` | Sufficient statistics and leaf parameter updates for a single tree |
| `StanBirthDeath` | Birth/death (grow/prune) MCMC proposals for a single tree |
| `StanForestFunctions` | Forest-level split variable sampling, including DART (Dirichlet) |
| `StanForest` | The full BART ensemble: manages `m` trees, runs one Gibbs sweep |
| `ForestEngine` | Thin wrapper around `StanForest`; the interface used by higher-level samplers |
| `OuterGibbsFunctions` | Outer Gibbs steps shared across models: `UpdateSigma`, `AugmentCensoredObservations`, `UpdateInvariantCoding` |
| `RcppExports.cpp` | Auto-generated by `Rcpp::compileAttributes()`; do not edit by hand |

### Include hierarchy (simplified)

```
Prerequisites.h  ←  Random.h
     ↑
   Info.h
     ↑
 StanTree.h  ←  StanTreeFunctions.h  ←  StanForestFunctions.h  ←  StanBirthDeath.h
                                                   ↑
                                             StanForest.h
                                                   ↑
                                            ForestEngine.h
```

`OuterGibbsFunctions.h` depends only on `Prerequisites.h` and is independent of the tree/forest hierarchy.

## Deprecated implementation (`deprecated/`)

The `deprecated/` subdirectory contains the **RJMCMC forest** — an earlier parallel implementation of BART that placed global-local shrinkage priors (horseshoe, half-Cauchy) directly on the leaf step heights. Because the step-height prior is no longer conjugate under these priors, a reversible-jump MCMC sampler was required, leading to a completely separate tree/forest stack.

This implementation was active while the package was called ShrinkageTrees / FusionTrees. It has been **deprecated** as FusionForests moves to the Stan-based engine for all models. The files are kept for reference.

### Deprecated file overview

| File(s) | Role |
|---|---|
| `Parameters` | Multi-parameter node representation (global shrinkage + local shrinkage + step height) |
| `Information` | `Data`, `Cutpoints`, `TreePrior` structs (built on top of `Parameters`) |
| `ScaleMixture` | Scale mixture prior: horseshoe, half-Cauchy, horseshoe-fw |
| `Tree` | RJMCMC tree node (carries a `Parameters` object and a `ScaleMixture`) |
| `Functions` | Tree utilities: splittable nodes, sufficient statistics, grow/prune probabilities |
| `TreeModifications` | Reversible-jump grow / prune / change / swap moves |
| `Forest` | Full RJMCMC ensemble; drives the `UpdateForest` Gibbs sweep |
| `TruncatedNormal` | Truncated normal sampler used by the probit and horseshoe models |
| `HorseTrees` | R entry point — single-forest horseshoe BART |
| `probit-HorseTrees` | R entry point — probit horseshoe BART |
| `CausalHorseForest` | R entry point — two-forest causal horseshoe BART (BCF-style) |

Files in `deprecated/` are **not compiled** by R's default build system (only the top-level `src/` directory is compiled) and are not registered in `RcppExports.cpp`.
